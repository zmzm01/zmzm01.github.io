Git 是什么

Git 是一个免费且开源的分布式版本控制系统，由 Linus Torvalds 创建，最初用于管理 Linux 内核的开发。它的核心作用包括：

  • 版本控制：Git 允许你跟踪代码或文本文件的历史变更，每次的改动都会被记录下来，可以轻松回溯到任何一个历史版本。
  • 协作开发：它为多个开发者同时在一个项目上工作提供了便利，每个人可以在本地修改代码，然后将更改合并到主线或共享分支中。
  • 分支管理：Git 的分支功能强大，使得并行开发和实验新特性变得非常灵活和高效。
  • 数据备份：通过将代码仓库托管在本地或远程服务器上，Git 确保了代码的安全性，避免了数据丢失的风险。

对于文本文件，Git 通过记录不同版本文件的“差异”完成对历史版本的记录，避免了重复储存完整文件。对于二进制文件，由于“差异”难以记录，Git 完整记录文件的不同历史版本。常见的文本文件有：.txt .rtf和各种源代码文件；常见的二进制文件有：.pdf .docx .pptx .xlsx等。

Git 基本命令

git的功能丰富而复杂，下面介绍了一些常用的git命令。

git init

初始化一个仓库，开始使用 Git 功能。

在Windows上，可以直接打开一个文件夹，右键选择Open Git Bash Here，然后直接git init即可。

git status

查看当前仓库状态，十分常用的命令。

git add

向暂存区中添加文件。要想让文件成为 Git 仓库的管理对象，就需要用 git add 命令将其加入暂存区（Stage 或者 Index）中。

使用touch命令只是用 Git 仓库的工作树创建了文件，该文件并不会被记入 Git 仓库的版本管理对象当中。

git commit

git commit 命令可以将当前暂存区中的文件实际保存到仓库的历史记录中。通过这些记录，我们就可以在工作树中复原文件。

git commit 需要添加提交信息，如果只有一行，可以使用：

-m后面连的就是需要提交的信息。

如果需要提交详细信息，直接执行git commit，根据 bash 的提示输入即可。

git log

git log 命令可以查看以往仓库中提交的日志。包括可以查看什么人在什么时候进行了提交或合并，以及操作前后有怎样的差别。

使用$ git log --pretty=short可以让程序只显示第一行简述信息。

使用$ git log path，path是目录或文件的路径，可以只显示该目录下的日志或只显示与该文件相关的日志。

使用$ git log -p可以查看文件的改动。

git diff

git diff 命令可以查看工作树、暂存区、最新提交之间的差别。

git branch

分支，顾名思义，就是不同的仓库状态。当你创建了一个分支，git 就在当前所在分支的基础上开始另一个分支，在新创建的分支上进行的操作是独立的，不会影响到原来的分支。这允许使用者同时进行多个功能的开发。

创建分支并切换到该分支：

或者简写为$ git checkout -b feature-A。

git 默认的分支是master分支，通常作为主干分支。

$ git branch -a可以同时查看本地仓库和远程仓库的分支。

git merge

使用命令$ git merge [branch]合并分支到当前所在分支。

如果存在冲突，git 似乎会默认删除当前所在分支存在冲突的部分。

添加--no-ff参数可以在历史记录中明确记录下本次分支合并：$ git merge --no-ff [branch]。

git log --graph可以以图表形式查看分支。

git reset

git 提供回溯历史版本的功能，每一个仓库状态对应一个唯一的哈希值，找到这个哈希值，使用命令$ git reset --hard [hash num]即可回溯到该状态。

使用$ git log可以查看以当前状态为终点的历史记录，从而查看哈希值。使用$ git log后按q退出。

要查看所有的历史操作，使用$ git reflog命令。

使用git commit --amend可以修改上一次的提交信息。

git rebase

该命令可以合并历史记录。

可以合并最新的两个历史记录。

git remote add

添加远程仓库。

这条命令将 GitHub 上的git@github.com:username/reponame.git仓库添加到远程仓库中，并将之标记为origin，以后可以用origin代指这个仓库。

git push

想将当前分支下本地仓库中的内容推送给远程仓库的特定分支。

-u 参数可以在推送的同时，将 origin 仓库的 master 分支设置为本地仓库当前分支的 upstream（上游）。添加了这个参数，将来运行 git pull 命令从远程仓库获取内容时，本地仓库的这个分支就可以直接从 origin 的 master 分支获取内容，省去了另外添加参数的麻烦。

git clone

执行 git clone 命令后我们会默认处于 master 分支下，同时系统会自动将 origin 设置成该远程仓库的标识符。

可以将 feature-D 分支克隆到本地的 feature-D 分支。

git pull

该命令可以将 origin 仓库中的 feature-D 分支的最新状态克隆到当前分支。
